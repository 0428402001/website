Downloading ZMap

The latest stable release of ZMap is version 1.0.3, which can be downloaded at v1.0.3.tar.gz.

ZMap is hosted on GitHub at https://github.com/zmap/zmap. The latest bleeding-edge version of ZMap can cloned from the git repository at git://github.com/zmap/zmap.git.

Issues should be reported on the Github Issue Tracker or sent to our team at zmap-team@umich.edu.

Building and Installing ZMap

ZMap is developed for use on Linux on 32-bit and 64-bit systems and can be built with most recent versions of gcc and the CMake build system. ZMap requires GMP, libpcap, Gengetopt, byacc, and flex. These can be installed as follows:

Debian or Ubuntu: sudo apt-get install libgmp3-dev libpcap-dev gengetopt byacc flex

Fedora, Red Hat Enterprise Linux, or CentOS: sudo yum install gmp gmp-devel libpcap-devel gengetopt byacc flex

Once these requirements have been installed, ZMap can be installed by running cmake -DENABLE_HARDENING=ON followed by make and sudo make install.

Redis support is not enabled by default. If you want to use ZMap with Redis, you will first need to install Hiredis. Then, rebuild ZMap with the command cmake -DWITH_REDIS=ON. If you don't want to run ZMap as root, you can give the binary permission to use the raw network with setcap cap_net_raw=ep /usr/local/sbin/zmap


JSON support is not enabled by default. If you want to output to JSON, install JSON-C. Then rebuild ZMap with cmake -DWITH_JSON=ON.
